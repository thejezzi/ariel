# Information Architecture

This docs app follows a simple 3-column shell:

1. **Left:** hierarchy navigation
2. **Center:** article content
3. **Right:** section TOC

## Why this structure works

- left rail answers *where am I in the docs tree?*
- center answers *what is this topic?*
- right rail answers *where in this page am I?*

## Trade-offs

- very wide sidebars can reduce reading width
- very narrow sidebars break long labels
- card-heavy designs increase cognitive noise
