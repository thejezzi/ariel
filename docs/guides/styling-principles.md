# Styling Principles

A good docs layout is optimized for reading speed, not decoration.

## Rules of thumb

### 1. Keep type scale tight

Use 4–5 sizes across the whole UI:

| token | typical use |
|---|---|
| sm | labels, metadata |
| base | body text |
| lg | section headings |
| xl | page heading |

### 2. Prefer structure over effects

- separators > cards everywhere
- neutral spacing > giant container gaps
- consistent rhythm > visual surprises

### 3. Optimize for scanability

- short paragraphs
- visible section headings
- predictable nav labels
- stable table of contents
